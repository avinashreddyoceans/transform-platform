## Getting Started

### 1. Start the Application

```bash
# Start Docker dependencies
docker compose -f .docker/docker-compose.yml up -d

# Build and run the app
./gradlew :platform-api:bootRun
```

### 2. Verify Health

```bash
curl http://localhost:8080/actuator/health
```

### 3. Create Your First Spec

```bash
curl -X POST http://localhost:8080/api/v1/specs \
  -H "Content-Type: application/json" \
  -d '{
    "name": "my-csv-spec",
    "format": "CSV",
    "delimiter": ",",
    "hasHeader": true,
    "fields": [
      {
        "name": "id",
        "type": "STRING",
        "required": true
      }
    ]
  }'
```

### 4. Transform a File

```bash
curl -X POST http://localhost:8080/api/v1/transform/file-to-events \
  -F "specId=<spec-id-from-step-3>" \
  -F "kafkaTopic=my-topic" \
  -F "file=@my-file.csv"
```

## Key Concepts

### File Specifications

A **Spec** defines the structure of a file:

```json
{
  "name": "bank-transactions-v1",
  "format": "CSV",               // CSV | FIXED_WIDTH | XML
  "encoding": "UTF-8",
  "delimiter": ",",              // For CSV
  "hasHeader": true,
  "skipLinesCount": 0,
  "fields": [
    {
      "name": "transactionId",
      "type": "STRING",
      "columnName": "TXN_ID",
      "required": true,
      "maxLength": 36
    }
  ]
}
```

### Transformations

**Transform** a file into Kafka events:

```json
{
  "specId": "spec-123",
  "kafkaTopic": "transactions",
  "skipInvalidRecords": false,
  "file": "<binary CSV data>"
}
```

Returns:

```json
{
  "correlationId": "corr-456",
  "recordsProcessed": 1000,
  "recordsPublished": 998,
  "recordsFailed": 2,
  "duration": "2.5s"
}
```

### Validation & Correction

Specs can include **validation rules** and **correction rules**:

```json
{
  "validationRules": [
    {
      "fieldName": "amount",
      "type": "RANGE",
      "min": 0,
      "max": 999999,
      "severity": "ERROR"
    }
  ],
  "correctionRules": [
    {
      "fieldName": "date",
      "type": "FORMAT_DATE",
      "sourceFormat": "MM/dd/yyyy",
      "targetFormat": "yyyy-MM-dd"
    }
  ]
}
```

## Error Handling

All endpoints follow standard HTTP status codes:

| Code | Meaning |
|------|---------|
| 200 | Success |
| 201 | Created |
| 204 | No Content |
| 400 | Bad Request (validation error) |
| 404 | Not Found |
| 409 | Conflict (duplicate spec name) |
| 500 | Server Error |

Error response format:

```json
{
  "timestamp": "2025-01-15T10:30:00Z",
  "status": 400,
  "error": "Bad Request",
  "message": "Field 'delimiter' is required for CSV format",
  "path": "/api/v1/specs"
}
```

## Authentication & Security

Development environment has **no authentication** by default.

For production:
- Configure OAuth2 or API Keys in `application.yml`
- Use `X-Correlation-Id` header for request tracking
- All endpoints use HTTPS
- CORS configured for `localhost:3000/3001` and GitHub Pages

---

## Rate Limiting

No rate limiting in development. Production defaults:
- 100 requests per minute per IP
- 1 file max 100MB
- 10 concurrent transformations per API key

---

## Webhooks & Async

File transformations are **asynchronous**:

1. POST to `/api/v1/transform/file-to-events` returns immediately with `correlationId`
2. File parsing happens in background
3. Records published to Kafka asynchronously
4. Poll `GET /api/v1/transform/status/{correlationId}` for updates

## Integration Examples

### With cURL

```bash
# List specs
curl http://localhost:8080/api/v1/specs

# Create spec
curl -X POST http://localhost:8080/api/v1/specs \
  -H "Content-Type: application/json" \
  -d @spec.json

# Transform file
curl -X POST http://localhost:8080/api/v1/transform/file-to-events \
  -F "specId=123" \
  -F "kafkaTopic=records" \
  -F "file=@data.csv"
```

### With Python

```python
import requests

# Create spec
spec = {
    "name": "csv-spec",
    "format": "CSV",
    "delimiter": ",",
    "fields": [{"name": "id", "type": "STRING"}]
}
resp = requests.post(
    "http://localhost:8080/api/v1/specs",
    json=spec
)
spec_id = resp.json()["id"]

# Transform file
with open("data.csv", "rb") as f:
    resp = requests.post(
        "http://localhost:8080/api/v1/transform/file-to-events",
        data={"specId": spec_id, "kafkaTopic": "records"},
        files={"file": f}
    )
    print(resp.json())
```

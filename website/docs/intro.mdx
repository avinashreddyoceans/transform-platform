# Transform Platform

**Move data between file formats and event streams — reliably, at scale, without writing pipeline code.**

Transform Platform is a spec-driven transformation engine built on Kotlin and Spring Boot. Teams define a `FileSpec` — a JSON schema that describes a file format, its fields, business rules, and delivery targets — and the platform handles parsing, validation, correction, and delivery automatically. New file formats and integration routes are added through configuration, not code.

---

## How It Works

Transform Platform handles both directions of data movement from a single spec. Whether you're ingesting partner files into your event stream or generating output files from processed events, the same `FileSpec` drives the pipeline.

## Supported File Formats

<Tabs>
  <TabItem value="now" label="Available Now">

| Format | Type | Module |
|--------|------|--------|
| **CSV / Delimited** | Character-separated values, any delimiter | `platform-core` |
| **Fixed-Width** | Positional flat files (ACH, NACHA, Mainframe exports) | `platform-core` |
| **XML** | XPath-mapped field extraction | `platform-core` |

  </TabItem>
  <TabItem value="planned" label="Planned">

| Format | Type | Notes |
|--------|------|-------|
| **JSON** | Document and event payloads | Phase 2 |
| **NACHA** | ACH file generation (outbound) | Phase 2 |
| **ISO 20022** | SEPA / SWIFT XML messaging standard | Phase 2 |
| **SWIFT MT** | Legacy SWIFT message formats | Phase 3 |
| **EDI / AS2** | ANSI X12 and EDIFACT | Phase 3 |

  </TabItem>
</Tabs>

Adding a new format requires implementing `FileParser` for inbound or `FileGenerator` for outbound — both are single-method interfaces. See [Adding a Parser](./extending/adding-a-parser) for the step-by-step guide.

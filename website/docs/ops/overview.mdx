# Operations Manual

This manual is the day-to-day reference for the team operating the Transform Platform. It covers every observability tool in the local stack ‚Äî what it does, when to reach for it, and exactly what to type when something goes wrong.

---

## The Observability Stack at a Glance

| Tool | URL | Credentials | Purpose |
|------|-----|-------------|---------|
| **Prometheus** | [localhost:9090](http://localhost:9090) | None | Metrics storage and alerting |
| **Grafana** | [localhost:3001](http://localhost:3001) | `admin / admin` | Metrics dashboards |
| **Kibana** | [localhost:5601](http://localhost:5601) | None | Structured log search |
| **Jaeger** | [localhost:16686](http://localhost:16686) | None | Distributed tracing |
| **Kafka UI** | [localhost:8090](http://localhost:8090) | None | Kafka topic browser |

| **App Health** | [localhost:8080/actuator/health](http://localhost:8080/actuator/health) | None | Liveness / readiness probe |

‚îÇ

## The Three Signals and How They Connect

Every request handled by the platform emits all three signals, all linked by the same `traceId`:

```
HTTP Request arrives
       ‚îÇ
       ‚ñº
CorrelationIdFilter ‚Äî writes correlationId to MDC + response header
       ‚îÇ
       ‚ñº
TracingMdcFilter ‚Äî writes traceId + spanId to MDC (from OTel span)
       ‚îÇ
       ‚îú‚îÄ‚îÄ LOG LINE emitted  ‚Üí  Kibana  (contains traceId, correlationId, level, message)
       ‚îÇ
       ‚îú‚îÄ‚îÄ SPAN emitted      ‚Üí  Jaeger  (contains traceId, HTTP method, URI, duration)
       ‚îÇ
       ‚îî‚îÄ‚îÄ METRIC incremented‚Üí  Prometheus / Grafana  (http_server_requests_seconds)
```

**Typical debug workflow:**

1. Grafana shows a spike in HTTP 5xx errors ‚Üí note the time window.
2. Kibana: filter `level: "ERROR"` in that time window ‚Üí copy a `traceId`.
3. Jaeger: paste the `traceId` ‚Üí see every span that made up that request.

## Starting the Stack

```bash
# From the project root
docker compose -f .docker/docker-compose.yml up -d

# Check all containers are healthy
docker compose -f .docker/docker-compose.yml ps

# Tail logs from all containers
docker compose -f .docker/docker-compose.yml logs -f

# Reload Prometheus config without restart (after changing rules)
curl -X POST http://localhost:9090/-/reload
```

## Log Correlation Fields

Every structured log line written by the app contains these fields:

| Field | Example | Description |
|-------|---------|-------------|
| `traceId` | `4bf92f3577b34da6a3ce929d0e0e4736` | OTel trace ID ‚Äî use in Jaeger |
| `spanId` | `00f067aa0ba902b7` | OTel span ID |
| `correlationId` | `1de41fa4-3d2c-48e7-acc4-297f0800bc5b` | Per-request UUID in response header `X-Correlation-ID` |
| `level` | `ERROR` | Log level: TRACE / DEBUG / INFO / WARN / ERROR |
| `logger` | `c.t.api.service.TransformService` | Logger class |
| `message` | `Transform failed for specId=csv-to-json` | Log message |
| `@timestamp` | `2026-03-09T10:45:32.123Z` | UTC timestamp |
